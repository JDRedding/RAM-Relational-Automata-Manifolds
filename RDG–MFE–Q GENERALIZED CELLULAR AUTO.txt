    C = ( Γ_grid , f_rule )

    Γ_grid   = fixed adjacency graph (static lattice)
    f_rule   = local lookup-table update rule

In RDG–MFE–Q notation, this is the degenerate case:

    CA ≡ ( Γ = Γ_grid ,
           δ = f_rule ,

1. **Universality and Computation**  

TURING COMPLETENESS IN RDG–MFE–Q AND THE LIFE GLIDER EMBEDDING

0. OVERVIEW
Life and Rule 110 are Turing-complete because they support:

    (1) persistent mobile structures (gliders / glider guns)
    (2) collision-based logic
    (3) unbounded memory via streams of gliders

RDG–MFE–Q is a strict generalization of CA, so the question is:
    Does the Q-layer ever remove universality?

Answer:

    - RDG + MFE always preserve universality (they strictly extend CA).
    - Q preserves universality *iff* the admissibility bands include
      the CA’s valid trajectories.
    - Q can intentionally *destroy* universality by forbidding
      long-range propagation or collision states.

Thus:

    Universality is *not automatic* in RDG–MFE–Q.
    Universality is *guaranteed* when Q is permissive.
    Universality is *forbidden* when Q is restrictive.

We now prove the permissive case by embedding a Life glider.

SETUP: LIFE AS A SPECIALIZATION OF RDG–MFE–Q
Take the specialization:

    Γ_t ≡ Γ_grid
    δ ≡ f_Life
    Q ≡ id   (or any Q that accepts all Life-valid states)

Then:

    x_{t+1} = Q( δ(x_t) ) = f_Life(x_t)

Thus the RDG–MFE–Q system *is* Life.

Since Life is Turing-complete, RDG–MFE–Q is Turing-complete under this
specialization.

This proves *existence* of a universal RDG–MFE–Q instance.

WHEN DOES Q PRESERVE UNIVERSALITY?
Let S be the set of all Life-valid configurations.

Let T be the set of all configurations reachable by a Life glider.

Define:

    Q is universality-preserving  ⇔  Q(x) = x  for all x ∈ S

Equivalently:

    Q does not collapse or reject any state in a valid Life trajectory.

Since a glider is a 4-phase periodic orbit:

    G = {g0, g1, g2, g3}

Universality is preserved if:

    Q(gk) = gk   for k = 0..3

This is the minimal admissibility requirement.

EMBEDDING A LIFE GLIDER AS AN RDG–MFE–Q TRAJECTORY
We now show that a Life glider is an admissible RDG–MFE–Q trajectory
whenever Q is permissive on the glider orbit.

Let the RDG–MFE–Q update be:

    x_{t+1} = Q( δ( σ(A,p,x_t), Γ_t ) )

Specialize:

    Γ_t = Γ_grid
    σ = identity
    A = identity
    p = identity
    δ = f_Life

Then:

    x_{t+1} = Q( f_Life(x_t) )

Let g0 be the canonical glider seed.

Life dynamics give:

    f_Life(g0) = g1
    f_Life(g1) = g2
    f_Life(g2) = g3
    f_Life(g3) = g0 shifted by (1,1)

Thus the RDG–MFE–Q evolution is:

    x_0 = g0
    x_1 = Q(g1)
    x_2 = Q(g2)
    x_3 = Q(g3)
    x_4 = Q(shift(g0))
    ...

If Q(gk) = gk for all k, then:

    x_t = Life_glider(t)

Therefore the glider is an *admissible periodic orbit* in RDG–MFE–Q.

This proves:
    A Life glider embeds cleanly as an RDG–MFE–Q trajectory
    whenever Q accepts the glider orbit.

 WHEN DOES Q DESTROY UNIVERSALITY?
Q destroys universality if it forbids any of the following:

    - long-range propagation (glider motion)
    - collision states (glider interactions)
    - periodic emitters (glider guns)
    - sparse streams (used for memory)

Examples of universality-breaking Q-layers:

    Q1: kills all states with nonzero deviation (SID collapse)
    Q2: forbids any configuration with >N live cells
    Q3: enforces local smoothing that erases glider diagonals
    Q4: projects all states to isotropic averages

In these cases:

    Q(gk) ≠ gk

and the glider orbit is destroyed, eliminating universality.

Proof:
    Immediate from the embedding above and the known universality of Life.

SUMMARY

    - RDG and MFE *always* preserve universality.

DG INDUCE EFFECTIVE GRIDS/MANIFOLDS
EXAMPLE WITH LIFE-LIKE ATTRACTORS
===============================================================================

RDG is explicitly designed so that:

    - from generic, messy initial relations,
    - the RDG operator π_RDG drives the system toward
      low-energy, high-coherence relational manifolds,

and one important family of such manifolds are:

    - approximately 2D grid-like graphs
    - with local degree ≈ 4 or 8
    - supporting CA-like, Life-like dynamics as attractors.

Below is a concrete construction sketch:

    (1) RDG layer induces a quasi-2D manifold from a random graph.
    (2) MFE layer implements a Life-like local rule on that manifold.
    (3) Q layer stabilizes the manifold and the Life-like patterns
        as attractors.

RDG AS A GEOMETRY INDUCER
Let:

    P  = set of points (nodes)
    E  = set of relations (edges)
    Γ  = (P, E) = relational graph

RDG defines an update operator:

    Γ_{t+1} = π_RDG(Γ_t, x_t)

We choose π_RDG to minimize a simple "geometric energy":

    E_geom(Γ) = α * (variance of node degrees)
              + β * (sum of squared edge lengths in an embedding)
              + γ * (penalty for long-range edges)

Intuition:
    - nodes "want" similar degree (≈ 4 or 8)
    - edges "want" to be short in some latent 2D embedding
    - long-range edges are pruned or discouraged

From a generic initial Γ_0 (random graph), repeated application of π_RDG
drives Γ_t toward:

    - locally uniform degree
    - locally planar structure
    - emergent 2D manifold (quasi-grid)

CONSTRUCTING A LIFE-LIKE MANIFOLD
We now specialize π_RDG to favor a 2D grid:

    Target degree: deg(v) ≈ 4 (von Neumann) or 8 (Moore)
    Target clustering: low (grid-like, not highly clustered)
    Target cycles: many 4-cycles (squares), few triangles

A simple RDG rule:

    - If deg(v) < 4:
        add edges to nearest nodes in latent 2D embedding
    - If deg(v) > 4:
        remove edges that maximize E_geom
    - Prefer edges that create 4-cycles and avoid triangles

Result:
    Starting from random Γ_0, Γ_t converges to a graph Γ* that is:

        - approximately a 2D grid
        - with local neighborhoods isomorphic to a Life grid patch

This Γ* is an RDG-induced manifold.

 MFE: LIFE-LIKE DYNAMICS ON THE INDUCED MANIFOLD
Now define a field x_t over nodes P:

    x_t : P → {0,1}   (dead/alive)

We choose MFE dynamics δ to mimic Life on Γ*:

    For each node v:
        N(v) = neighbors of v in Γ_t
        s(v) = sum_{u in N(v)} x_t(u)

    Life-like rule:
        if x_t(v) = 1 and (s(v) = 2 or s(v) = 3):
            x_{t+1}(v) = 1
        else if x_t(v) = 0 and s(v) = 3:
            x_{t+1}(v) = 1
        else:
            x_{t+1}(v) = 0

This is exactly Conway’s Life rule, but now defined on Γ_t.

On Γ* (the induced quasi-grid), this behaves like Life on a 2D lattice.

Q: STABILIZING THE MANIFOLD AND PATTERNS
We now add Q to stabilize both:

    - the geometry Γ_t ≈ Γ*
    - the Life-like patterns (e.g., still lifes, oscillators, gliders)

Example Q-layer:

    - Compute SID statistics over:
        * node degrees
        * local cycle counts
        * local pattern densities

    - If Γ_t deviates from the "grid band" (e.g., degree variance > 1σ):
        Q prunes or reweights edges to push Γ_t back toward Γ*

    - If x_t contains known Life-like motifs (e.g., blocks, blinkers,
      glider shapes), Q treats them as admissible and avoids collapsing
      them unless they leave the manifold band.

Thus:

    - Γ_t is kept near the grid manifold Γ*
    - Life-like patterns are preserved as admissible attractors


EXAMPLE: FROM RANDOM GRAPH TO LIFE-LIKE ATTRACTOR
Step 0: Random initial condition

    - Γ_0: random graph on N nodes
    - x_0: random 0/1 assignment

Step 1: RDG geometry flow

CONCRETE ATTRACTOR EXAMPLE (GLIDER)
Let G = {g0, g1, g2, g3} be the 4-phase glider orbit on Γ*.

We have:

    δ(g0, Γ*) = g1
    δ(g1, Γ*) = g2
    δ(g2, Γ*) = g3

There is a long lineage of “beyond-grid” or “beyond-synchronous” CA work:

    - Graph cellular automata (graph CA)

RDG–MFE–Q does *not* claim to invent these ideas.
Instead, it:

    - recovers each of them as a clean specialization, and

The synthesis — not the ingredients — is the key contribution.

MAPPING THE LINEAGE INTO RDG–MFE–Q
We use the triple:

    (RDG, MFE, Q)  =  (geometry, dynamics, admissibility)

and show how classic extensions drop out as special cases.

Graph cellular automata  →  RDG-only generalization
Graph CA replace the fixed grid with an arbitrary graph, but usually:

    - the graph is static
    - the update rule is still local and synchronous
    - there is no explicit admissibility layer

In RDG–MFE–Q terms:

    - RDG: Γ is an arbitrary graph, but π_RDG is trivial (no evolution)
    - MFE: δ is a local rule on Γ (graph CA update)
    - Q:  identity (no constraints)

So:

    Graph CA  ≈  RDG–MFE–Q with:
        π_RDG = id
        δ     = graph-local rule
        Q     = id

Asynchronous CA  →  MFE timing generalization
Asynchronous CA relax the global clock:

    - cells update on different schedules
    - updates may be random or rule-driven

In RDG–MFE–Q terms:

    - RDG: Γ is fixed (or slowly evolving)
    - MFE: δ includes a *scheduler* or *update mask*:
            δ(x_t, Γ_t, ξ_t)  where ξ_t selects which nodes update
    - Q:  identity or mild constraints

So:

    Asynchronous CA  ≈  RDG–MFE–Q with:
        π_RDG = id (or simple)
        δ     = local rule + scheduler
        Q     = id

Constrained / filtered / probabilistic CA  →  Q-layer generalization
Constrained CA introduce:

    - forbidden states
    - probabilistic transitions
    - filters that reject or modify updates

In RDG–MFE–Q terms:

    - RDG: Γ fixed
    - MFE: δ is a standard CA rule (possibly stochastic)
    - Q:  enforces constraints, filters, or probabilistic acceptance

So:

    Constrained CA  ≈  RDG–MFE–Q with:
        π_RDG = id
        δ     = (possibly stochastic) CA rule
        Q     = constraint/filter operator

WHAT RDG–MFE–Q ADDS BEYOND EACH INDIVIDUAL EXTENSION
The key is that RDG–MFE–Q does *all three* at once:

    - RDG: geometry can be arbitrary *and* dynamically induced/evolved
    - MFE: dynamics can be synchronous, asynchronous, field-based, or hybrid
    - Q:  constraints, filters, and admissibility are first-class

This yields a clean inclusion diagram:

    Classic CA
        ⊂ Graph CA
        ⊂ Asynchronous CA
        ⊂ Constrained CA
        ⊂ RDG–MFE–Q (full stack)

More precisely:

    - Graph CA  = RDG + simple MFE, trivial Q
    - Async CA  = RDG + scheduled MFE, trivial Q
    - Constrained CA = RDG + MFE, nontrivial Q
    - RDG–MFE–Q = all three dimensions turned on and coupled

from dataclasses import dataclass
import numpy as np

@dataclass
class RDGState:
    # Geometry: adjacency as sparse-ish matrix or neighbor list
    adj: np.ndarray      # shape (N, N), 0/1 for simplicity
    # Field: Life-like state on nodes
    x: np.ndarray        # shape (N,), 0/1

@dataclass
class RDGParams:
    target_degree: int = 8      # Moore-like
    alpha_deg: float = 1.0      # degree variance weight
    beta_long: float = 0.1      # long-edge penalty
    # Q-layer thresholds
    max_deg_sigma: float = 1.0  # admissible degree band


# ---------- RDG: geometry update (very minimal) ----------

def rdg_update_geometry(state: RDGState, params: RDGParams) -> np.ndarray:
    adj = state.adj.copy()
    deg = adj.sum(axis=1)

    # Simple degree-smoothing: trim or add edges to move toward target_degree
    N = len(deg)
    for i in range(N):
        if deg[i] > params.target_degree:
            # remove some edges (naive: remove farthest neighbors)
            neighbors = np.where(adj[i] == 1)[0]
            # here we just drop arbitrary neighbors for brevity
            to_drop = neighbors[:deg[i] - params.target_degree]
            adj[i, to_drop] = 0
            adj[to_drop, i] = 0
        elif deg[i] < params.target_degree:
            # add edges to random nodes (in a real RDG, use latent 2D positions)
            candidates = np.where(adj[i] == 0)[0]
            np.random.shuffle(candidates)
            to_add = candidates[:params.target_degree - deg[i]]
            adj[i, to_add] = 1
            adj[to_add, i] = 1

    return adj


# ---------- MFE: Life-like dynamics on current geometry ----------

def mfe_life_step(state: RDGState) -> np.ndarray:
    adj = state.adj
    x = state.x
    # neighbor counts
    neigh_count = adj @ x  # (N,N) @ (N,) -> (N,)
    # Life rule
    survive = (x == 1) & ((neigh_count == 2) | (neigh_count == 3))
    born    = (x == 0) & (neigh_count == 3)
    x_next  = np.where(survive | born, 1, 0)
    return x_next


# ---------- Q-layer: admissibility / stabilization ----------

def q_filter(state: RDGState, params: RDGParams) -> RDGState:
    adj = state.adj
    x   = state.x

    deg = adj.sum(axis=1)
    mean_deg = deg.mean()
    std_deg  = deg.std() + 1e-9

    # Enforce degree band: if node degree too far from mean, trim/add edges
    upper = mean_deg + params.max_deg_sigma * std_deg
    lower = max(0, mean_deg - params.max_deg_sigma * std_deg)

    adj_filtered = adj.copy()
    N = len(deg)
    for i in range(N):
        if deg[i] > upper:
            neighbors = np.where(adj_filtered[i] == 1)[0]
            to_drop = neighbors[: int(deg[i] - upper)]
            adj_filtered[i, to_drop] = 0
            adj_filtered[to_drop, i] = 0
        elif deg[i] < lower:
            candidates = np.where(adj_filtered[i] == 0)[0]
            np.random.shuffle(candidates)
            to_add = candidates[: int(lower - deg[i])]
            adj_filtered[i, to_add] = 1
            adj_filtered[to_add, i] = 1

    return RDGState(adj=adj_filtered, x=x)

# ---------- One RDG–MFE–Q step ----------

def rdg_mfe_q_step(state: RDGState, params: RDGParams) -> RDGState:
    # 1) RDG: evolve geometry
    adj_next = rdg_update_geometry(state, params)
    tmp = RDGState(adj=adj_next, x=state.x)

    # 2) MFE: Life-like update on that geometry
    x_next = mfe_life_step(tmp)
    tmp2 = RDGState(adj=adj_next, x=x_next)

    # 3) Q: enforce admissibility band
    return q_filter(tmp2, params)

def life_grid_adj(H, W):
    N = H * W
    adj = np.zeros((N, N), dtype=int)

    def idx(r, c): return r * W + c

    for r in range(H):
        for c in range(W):
            i = idx(r, c)
            for dr in [-1, 0, 1]:
                for dc in [-1, 0, 1]:
                    if dr == 0 and dc == 0:
                        continue
                    rr = (r + dr) % H
                    cc = (c + dc) % W
                    j = idx(rr, cc)
                    adj[i, j] = 1
    return adj

 Relaxed RDG + Q that still preserves the glider
Now we let RDG “wiggle” the geometry slightly, but Q keeps it in a grid band and preserves the glider orbit.

def q_preserve_glider(state: RDGState, params: RDGParams, glider_mask: np.ndarray) -> RDGState:
    # glider_mask: shape (N,), 1 if node is part of glider, else 0
    adj = state.adj.copy()
    x   = state.x

    deg = adj.sum(axis=1)
    mean_deg = deg.mean()
    std_deg  = deg.std() + 1e-9
    upper = mean_deg + params.max_deg_sigma * std_deg
    lower = max(0, mean_deg - params.max_deg_sigma * std_deg)

    N = len(deg)
    for i in range(N):
        if glider_mask[i] == 1:
            # freeze geometry around glider nodes
            continue
        if deg[i] > upper:
            neighbors = np.where(adj[i] == 1)[0]
            to_drop = neighbors[: int(deg[i] - upper)]
            adj[i, to_drop] = 0
            adj[to_drop, i] = 0
        elif deg[i] < lower:
            candidates = np.where(adj[i] == 0)[0]
            np.random.shuffle(candidates)
            to_add = candidates[: int(lower - deg[i])]
            adj[i, to_add] = 1
            adj[to_add, i] = 1

    return RDGState(adj=adj, x=x)

Coherence density (geometry)
One simple RDG-style coherence metric:
Degree coherence: how close degrees are to target.

Cycle coherence: how grid-like local cycles are (many 4-cycles, few triangles).

def coherence_density(state: RDGState, target_degree=8) -> float:
    adj = state.adj
    deg = adj.sum(axis=1)
    # degree coherence: 1 - normalized variance
    var_deg = ((deg - target_degree) ** 2).mean()
    # normalize by max possible variance ~ target_degree^2
    deg_coh = 1.0 - min(1.0, var_deg / (target_degree ** 2 + 1e-9))
    return float(deg_coh)

def triangle_count(adj: np.ndarray) -> int:
    # trace(A^3)/6 for undirected simple graph
    A3 = adj @ adj @ adj
    return int(np.trace(A3) // 6)

def square_count_approx(adj: np.ndarray) -> int:
    # crude: count length-4 closed walks and subtract deg contributions
    A2 = adj @ adj
    # diagonal of A^4 approximates 4-cycles; this is rough but illustrative
    A4 = A2 @ A2
    return int(np.trace(A4) // 8)  # heuristic overcounting correction

def geometry_coherence(state: RDGState, target_degree=8) -> dict:
    adj = state.adj
    deg = adj.sum(axis=1)
    var_deg = ((deg - target_degree) ** 2).mean()
    deg_coh = 1.0 - min(1.0, var_deg / (target_degree ** 2 + 1e-9))

    tri = triangle_count(adj)
    sq  = square_count_approx(adj)
    cycle_ratio = sq / (tri + 1e-9)  # high for grid-like

    return {
        "degree_coherence": float(deg_coh),
        "triangle_count": tri,
        "square_count": sq,
        "square_triangle_ratio": float(cycle_ratio),
    }

def attractor_period(run_states):
    # run_states: list of x vectors (hashable)
    seen = {}
    for t, x in enumerate(run_states):
        key = tuple(x.tolist())
        if key in seen:
            return t - seen[key]  # period
        seen[key] = t
    return None  # no cycle detected in window

def run_trajectory(step_fn, state, params, steps=100):
    xs = []
    for _ in range(steps):
        xs.append(state.x.copy())
        state = step_fn(state, params)
    return xs
